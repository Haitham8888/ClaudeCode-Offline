Claude Code Offline

تشغيل Claude Code بدون إنترنت وبدون حساب عبر SGLang + DeepSeek V4 Flash
Offline v2.1.170 Windows Linux

بدء سريع

لتشغيل Claude Code أوفلاين بالكامل، تحتاج فقط ثلاثة متغيرات بيئية وسيرفر SGLang يعمل على المنفذ 30000:

@echo off
set ANTHROPIC_BASE_URL=http://TBD:30000
set ANTHROPIC_API_KEY=sk-offline
set ANTHROPIC_DEFAULT_HAIKU_MODEL=DeepSeek-V4-Flash
set CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1

claude --model DeepSeek-V4-Flash
$env:ANTHROPIC_BASE_URL = "http://TBD:30000"
$env:ANTHROPIC_API_KEY = "sk-offline"
$env:ANTHROPIC_DEFAULT_HAIKU_MODEL = "DeepSeek-V4-Flash"
$env:CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC = "1"

claude --model DeepSeek-V4-Flash
export ANTHROPIC_BASE_URL=http://TBD:30000
export ANTHROPIC_API_KEY=sk-offline
export ANTHROPIC_DEFAULT_HAIKU_MODEL=DeepSeek-V4-Flash
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1

claude --model DeepSeek-V4-Flash

قم بتغيير TBD إلى عنوان IP الخاص بسيرفر SGLang الفعلي.

تم التحقق: Claude Code v2.1.170 يعمل بهذه الطريقة بدون أي تسجيل دخول وبدون إنترنت، ويدعم REPL التفاعلي والأوامر النصية و /agents.

المتطلبات الكاملة

A. السيرفر — شغال وجاهز (RHEL 10 + SGLang + DeepSeek V4 Flash)

السيرفر شغال مسبقا ولا يحتاج أي تغيير:

  • المعالج: 4x H100 94GB GPUs
  • النظام: RHEL 10 مع Podman + CUDA 13
  • SGLang: شغال على v0.5.12.post1 مع /v1/messages endpoint على منفذ 30000
  • الموديل: DeepSeek V4 Flash محمّل مع --tool-call-parser deepseekv3
  • IP: متاح على http://TBD:30000 على الشبكة المحلية
السيرفر جاهز بالكامل. هذه الحزمة تخص الجهاز العميل فقط. لا تحتاج تثبيت SGLang أو تحميل الموديل — كل شيء موجود.

الإعدادات والتخصيص

ملف الإعدادات الدائم (~/.claude/settings.json)

يمكنك وضع جميع المتغيرات البيئية في ملف الإعدادات لتثبيتها بشكل دائم، بحيث لا تحتاج لتعيينها كل مرة:

{
  "permissions": {
    "allow": ["Bash(*)", "Read(*)", "Write(*)", "Edit(*)"]
  },
  "env": {
    "ANTHROPIC_BASE_URL": "http://TBD:30000",
    "ANTHROPIC_API_KEY": "sk-offline",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "DeepSeek-V4-Flash",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "DISABLE_AUTOUPDATER": "1",
    "CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY": "1"
  },
  "fallbackModel": [
    "DeepSeek-V4-Flash"
  ]
}

هذا الملف موجود مسبقا في config/settings.json - فقط غير IP السيرفر.

ملاحظة: يمكنك استخدام متغيرات البيئة بدلا من الملف. الأولوية لمتغيرات البيئة إذا كانت موجودة.

جميع المتغيرات البيئية

المتغيرإلزاميالوظيفة
ANTHROPIC_BASE_URLنعم يعيد توجيه جميع طلبات Claude Code من خوادم Anthropic إلى سيرفر SGLang المحلي. هذا هو المفتاح الأساسي للتشغيل الأوفلاين.
ANTHROPIC_API_KEYنعم مفتاح API وهمي (مثلاً sk-offline). SGLang لا يتحقق من صحته. ضروري لمنع Claude Code من طلب تسجيل الدخول.
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFICموصى به يمنع جميع الاتصالات غير الضرورية: التليمتري، تقارير الأخطاء، التحديثات، الاستبيانات، أي طلبات HTTP غير متعلقة بالـ API.
DISABLE_AUTOUPDATERموصى به يعطل فحص التحديثات التلقائي.
DISABLE_TELEMETRYاختياري يمنع إرسال بيانات الاستخدام.
DO_NOT_TRACKاختياري يعادل DISABLE_TELEMETRY.
CLAUDE_CODE_DISABLE_OFFICIAL_MARKETPLACE_AUTOINSTALLاختياري يمنع التثبيت التلقائي لإضافات السوق الرسمي.
CLAUDE_CODE_DISABLE_BACKGROUND_TASKSاختياري يمنع المهام الخلفية.
DISABLE_LOGIN_COMMANDاختياري يخفي أمر /login بالكامل.
CLAUDE_CODE_SIMPLEاختياري يعطل auto-discovery لل hooks وال plugins و MCP servers و skills. يفعّل وضع --print تلقائيا.
ANTHROPIC_DEFAULT_HAIKU_MODELموصى به يحدد الموديل المستخدم للمهام الخلفية الصغيرة (تسمية الجلسات، التلخيص، تسمية الوكلاء). مهم جدا عند تشغيل وكلاء متعددين أوفلاين — بدونها سيحاول Claude Code استخدام موديل Haiku من خوادم Anthropic ويفشل. اضبطه على نفس اسم موديل DeepSeek المحلي. (البديل: ANTHROPIC_SMALL_FAST_MODEL)
ANTHROPIC_SMALL_FAST_MODELاختياري بديل عن ANTHROPIC_DEFAULT_HAIKU_MODEL في الإصدارات الأقدم. يتحكم بنفس الوظيفة: الموديل المستخدم للمهام الخلفية.
CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERYاختياري فعّل باكتشاف الموديلات من سيرفر SGLang المحلي. عندما =1، يملأ Claude Code قائمة /model من endpoint /v1/models على السيرفر المحلي. مفيد إذا كان السيرفر يقدم عدة موديلات.
MAX_THINKING_TOKENSاختياري التحكم في الـ thinking tokens. اضبط =0 لتعطيل extended thinking تماما (مفيد مع الموديلات المحلية التي لا تدعم thinking).
ENABLE_TOOL_SEARCHاختياري التحكم في بحث أدوات MCP. القيمة false تحمّل جميع الأدوات مسبقا. القيمة true تفعّل التحميل المؤجل (deferred). عندما تستخدم ANTHROPIC_BASE_URL لسيرفر غير تابع لـ Anthropic، هذا المتغير يتحكم في سلوك الأدوات.

إعدادات إضافية في settings.json (v2.1.166+)

بالإضافة إلى المتغيرات البيئية، يمكنك استخدام هذه الإعدادات في ملف ~/.claude/settings.json:

الإعدادالوظيفة
fallbackModel يحدد حتى 3 موديلات احتياطية (fallback) عندما يكون الموديل الأساسي غير متاح أو محملا فوق طاقته. مفيد جدا للبيئة المحلية: إذا كان DeepSeek V4 Flash مشغولا، يمكنك تعيين موديل احتياطي آخر على نفس سيرفر SGLang. مثال: 
"fallbackModel": [
  "DeepSeek-V4-Flash",
  "DeepSeek-V3-Lite",
  "Qwen3-Coder-32B"
]
disableBundledSkills يعطل الـ skills المضمنة والأوامر slash commands المدمجة. مثال: "disableBundledSkills": true. (يتطلب v2.1.169+)

طرق التشغيل

الطريقةالأمرالوصف
وضع تفاعلي (REPL) claude --model DeepSeek-V4-Flash جلسة كاملة متعددة الأدوار. الأنسب للتطوير والبرمجة.
أمر لمرة واحدة claude --bare -p "..." سؤال واحد مع رد مباشر. مناسب للسكربتات وخطوط الأنابيب.
إدخال من STDIN echo "..." | claude --bare تمرير النص عبر pipe.
وضع بدون أذونات claude --model DeepSeek-V4-Flash --permission-mode bypassPermissions ينفذ الأوامر دون طلب موافقة لكل إجراء.
ملاحظة: --bare يعطل الميزات المتقدمة مثل hooks و LSP و plugin sync و keychain reads. إذا احتجت هذه الميزات، شغل بدون --bare.

التحقق من صحة العمل

لتتأكد أن Claude Code يعمل بشكل صحيح مع سيرفر SGLang المحلي، اتبع هذه الخطوات:

1. التحقق من اتصال السيرفر

تأكد أن سيرفر SGLang يستجيب:

# اختبار قائمة الموديلات
curl http://TBD:30000/v1/models

# يجب أن ترى رد مثل:
{
  "object": "list",
  "data": [
    { "id": "DeepSeek-V4-Flash", "object": "model" }
  ]
}

2. التحقق من إصدار Claude Code

# تأكد أن الأمر معروف
claude --version

# النتيجة المتوقعة:
2.1.170

3. اختبار أمر لمرة واحدة

# شغل مع متغيرات البيئة
set ANTHROPIC_BASE_URL=http://TBD:30000
set ANTHROPIC_API_KEY=sk-offline
set ANTHROPIC_DEFAULT_HAIKU_MODEL=DeepSeek-V4-Flash
set CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
claude --bare -p "اكتب كلمة مرحبا"

# إذا رأيت ردا (وليس خطأ اتصال) فالعمل صحيح.

4. اختبار الوضع التفاعلي

# في نافذة تيرمينال جديدة
set ANTHROPIC_BASE_URL=http://TBD:30000
set ANTHROPIC_API_KEY=sk-offline
set ANTHROPIC_DEFAULT_HAIKU_MODEL=DeepSeek-V4-Flash
set CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
claude --model DeepSeek-V4-Flash

إذا فتح REPL وظهرت شاشة الترحيب، فكل شيء يعمل بشكل صحيح.

5. اختبار /agents

داخل جلسة Claude Code، اكتب /agents لعرض قائمة الوكلاء المتاحين. إذا ظهرت القائمة، فالميزة تعمل.

جميع الأوامر slash commands

الأوامر التالية متاحة داخل جلسة Claude Code. كلها تعمل أوفلاين (إما محليا أو عبر API السيرفر المحلي) باستثناء WebFetch و WebSearch اللذين يحتاجان اتصال إنترنت حقيقي.

الأمروظيفته واستخدامهالنوع
إدارة الجلسة
/clear مسح سياق المحادثة وبدء جلسة جديدة. الجلسة السابقة تبقى محفوظة على القرص ويمكن استئنافها لاحقا بـ /resume. إذا كنت تريد تحرير مساحة السياق مع الاستمرار في نفس المحادثة، استخدم /compact بدلا منه. (بدائل: /reset, /new) محلي
/branch [name] إنشاء نسخة متفرعة من المحادثة الحالية. يأخذ المحادثة كما هي وينسخها إلى جلسة جديدة منفصلة، ويتحول إليها مباشرة. الجلسة الأصلية تبقى كما هي ويمكن الرجوع إليها بـ /resume. مفيد عندما تريد تجربة اتجاه مختلف دون المخاطرة بفقدان المحادثة الأصلية. إذا كنت تريد تفويض مهمة جانبية لوكيل في الخلفية بدلا من التفرع بنفسك، استخدم /fork. (البديل: /fork في الإصدارات القديمة) محلي
/resume [session] استئناف جلسة سابقة. يمكنك تحديد الجلسة بواسطة معرفها أو اسمها، أو تركها فارغة لفتح منتقي الجلسات التفاعلي. جميع الجلسات المحفوظة (بما في ذلك جلسات الخلفية) تظهر في القائمة. (البديل: /continue) محلي
/background [prompt] فصل الجلسة الحالية لتعمل في الخلفية وتحرير التيرمينال. يمكنك تمرير تعليمة إضافية قبل الانفصال. لعرض وإدارة جلسات الخلفية، استخدم claude agents من تيرمينال آخر. الجلسات في الخلفية تظهر في /resume مع علامة "bg". (البديل: /bg) محلي
/compact [instructions] تلخيص المحادثة لتوفير مساحة السياق. يقوم Claude بتلخيص المحادثة حتى الآن مع الحفاظ على المعلومات المهمة. يمكنك تمرير تعليمات تركيز اختيارية، مثل: /compact focus on edited files and todos ليخبر Claude بما يجب الحفاظ عليه. يختلف عن /clear بأنه يستمر في نفس الجلسة بدلا من بدء جلسة جديدة. API
/context [all] عرض استهلاك السياق الحالي على شكل شبكة ملونة. يظهر كل عنصر في السياق (رسائل، ملفات، أدوات) وحجمه النسبي. في وضع الملء، تكون العناصر مطوية للحفاظ على الشبكة مرئية. استخدم /context all لتوسيع جميع العناصر ورؤية التفاصيل الكاملة. مفيد لمعرفة ما يستهلك مساحة السياق قبل استخدام /compact. محلي
/fork [directive] إنشاء وكيل فرعي في الخلفية يرث المحادثة الحالية. يعمل الوكيل على التعليمات التي تمررها أثناء استمرارك في المحادثة الأصلية. نتيجتها تعود إلى محادثتك عندما ينتهي. يختلف عن /branch بأن /branch يحولك أنت إلى نسخة، بينما /fork يفوض المهمة لوكيل آخر. (قبل v2.1.161 كان /fork مرادفا لـ /branch) API
الملفات والمشاريع
/add-dir <path> إضافة مجلد عمل إضافي للجلسة الحالية. يمنح Claude صلاحية الوصول لملفات المجلد المضاف. معظم إعدادات .claude/ لا تُكتشف من المجلد المضاف. يمكنك لاحقا استئناف الجلسة من المجلد المضاف باستخدام --continue أو --resume. محلي
/cd <path> نقل الجلسة إلى مجلد عمل جديد. ذاكرة المحادثة المخبأة تبقى محفوظة: ملف CLAUDE.md من المجلد الجديد يضاف كرسالة بدلا من إعادة بناء system prompt. الجلسة تنقل إلى مجلد المشروع الجديد، لذلك /resume و /continue سيجدانها هناك. إذا لم تثق بالمجلد من قبل، سيطلب منك تأكيد الثقة. (يتطلب v2.1.169+) محلي
/diff عرض التغييرات غير المرفوعة وفروقات كل جولة. يفتح عارضا تفاعليا. استخدم الأسهم اليمين/اليسار للتبديل بين diff الحالي وجولات Claude، والأسهم أعلى/أسفل لتصفح الملفات. مفيد قبل رفع التغييرات لمراجعة ما تم تغييره. محلي
/init تهيئة المشروع بملف CLAUDE.md. ينشئ ملف CLAUDE.md يحتوي على تعليمات للمشروع ليساعد Claude على فهم السياق. إذا كان CLAUDE_CODE_NEW_INIT=1، يفتح وضعا تفاعليا يرشدك خلال إعداد الملفات الشخصية والـ skills و hooks. محلي
الوكلاء (Agents)
/agents إدارة تكوينات الوكلاء (sub-agents). يفتح مدير الوكلاء لإنشاء وحذف وعرض وتشغيل وكلاء متخصصين. الوكلاء هم جلسات Claude Code مستقلة يمكنها التفويض بمهام جانبية (مثل مراجعة الكود أو كتابة الاختبارات). يمكن لكل وكيل أن يكون له شخصيته الخاصة وتعليماته المخصصة. جميع الوكلاء يستخدمون نفس API السيرفر المحلي. محلي + API
الاستعلام والمحادثة
/btw <question> طرح سؤال جانبي سريع دون مقاطعة المحادثة الرئيسية. يعمل عن طريق إنشاء وكيل مؤقت عابر بدون أدوات - لا يمكنه قراءة ملفات أو تشغيل أوامر. يرد مرة واحدة بناء على سياق المحادثة الحالي فقط، ثم يختفي. السؤال والرد لا يدخلان في تاريخ المحادثة الرئيسي أبدا. يمكنك أيضا كتابة "btw" (أحرف صغيرة) في بداية رسالتك وسيكتشفه Claude Code تلقائيا. API
/copy [N] نسخ آخر رد من المساعد إلى الحافظة. /copy 2 ينسخ الرد قبل الأخير. عندما يكون هناك code blocks، يظهر منتقي تفاعلي لاختيار مقتطف معين أو الرد كاملا. اضغط w في المنتقي لكتابة التحديد إلى ملف بدلا من الحافظة (مفيد عبر SSH). محلي
/rename <name> تسمية الجلسة الحالية. الاسم يظهر في /resume وفي عنوان التيرمينال. يمكنك استئناف جلسة مسماة باستخدام claude --resume <name>. محلي
الإعدادات والتخصيص
/config فتح واجهة الإعدادات لتعديل الثيم، الموديل، نمط الإخراج، والتفضيلات الأخرى. (البديل: /settings) محلي
/color [color|default] تغيير لون شريط الأوامر للجلسة. الألوان: red, blue, green, yellow, purple, orange, pink, cyan. /color default يعيد اللون الافتراضي. بدون وسيط يختار لونا عشوائيا. محلي
/doctor تشخيص التثبيت والإعدادات والتحقق من صحة العمل. يتحقق من البيئة والإعدادات والاتصال. النتائج تظهر مع أيقونات الحالة. اضغط f ليحاول Claude إصلاح أي مشاكل تلقائيا. يفضل تشغيله بعد التثبيت للتأكد من أن كل شيء يعمل. محلي + API
/help عرض قائمة الأوامر المتاحة. اكتب / في أي وقت داخل الجلسة لرؤية القائمة الكاملة. محلي
/release-notes عرض سجل التغييرات (changelog). يفتح منتقي إصدارات تفاعلي لتصفح ملاحظات الإصدار واختيار إصدار معين. محلي
الأدوات المضمنة (Tools)
Bash تنفيذ أوامر الشيل (PowerShell, CMD, Bash). يدعم timeout اختياري وتشغيل الأوامر في الخلفية. الأداة الأساسية لتفاعل Claude مع نظام التشغيل. محلي
Edit تعديل محتويات الملفات في مكانها. يستبدل نصوصا محددة داخل الملفات. يدعم replace_all لاستبدال جميع التكرارات. آمن لأنه يتطلب مطابقة تامة للنص القديم. محلي
Read قراءة الملفات والصور وملفات PDF و Jupyter Notebooks. يدعم قراءة أجزاء محددة باستخدام offset و limit. محلي
Glob البحث عن الملفات بأنماط glob. سريع في قواعد الأكواد الكبيرة. يدعم أنماطا مثل **/*.js. محلي
Grep البحث عن نصوص في محتويات الملفات باستخدام regex. يدعم regex كامل وتصفية الملفات بنمط include. محلي
TodoWrite إنشاء وإدارة قائمة مهام منظمة. يتتبع الخطوات المتعددة والمهام المعقدة مع تحديث الحالات والأولويات. محلي
LS سرد محتويات المجلدات. يعرض محتويات مجلد معين مع تمييز المجلدات الفرعية. محلي
NotebookRead قراءة ملفات Jupyter Notebook (.ipynb). يستخرج الخلايا والمخرجات من ملفات الدفاتر. محلي
WebFetch جلب محتوى من URL. يحول الصفحات إلى نص أو markdown. (لا يعمل بدون إنترنت) شبكة
WebSearch البحث في الويب. يبحث عن معلومات حديثة من الإنترنت. (لا يعمل بدون إنترنت) شبكة
تصنيف الأنواع: محلي - ينفذ بالكامل على جهازك دون أي اتصال خارجي. API - يحتاج الاتصال بسيرفر SGLang المحلي (وليس بالإنترنت). شبكة - يحتاج اتصال إنترنت حقيقي (لن يعمل في بيئة معزولة عن الإنترنت). فقط WebFetch و WebSearch من هذا النوع.

كيف يعمل النظام

  +-------------------------------------------------------+
  |                  Client Machine                        |
  |  +------------------+       +----------------------+  |
  |  |   Claude Code    | --->  |  SGLang Server       |  |
  |  |   (REPL / CLI)   |HTTP   |  (RHEL / Podman)     |  |
  |  |                  |:30000 |  /v1/messages        |  |
  |  |  Tools:          | <---  |  DeepSeek V4 Flash   |  |
  |  |  Bash, Edit,     | JSON  |  CUDA 13             |  |
  |  |  Read, Glob, ... |       +----------------------+  |
  |  +------------------+                                  |
  |         |                                               |
  |         v                                               |
  |  +------------------+                                   |
  |  |  Project Files   |  read/write locally              |
  |  +------------------+                                   |
  |                                                         |
  |  No Internet    No Login    No Telemetry                |
  +-------------------------------------------------------+

آلية العمل

  1. Claude Code يرسل طلب POST إلى /v1/messages على سيرفر SGLang المحلي
  2. SGLang يستقبل الطلب بصيغة Anthropic Messages API ويترجمه إلى صيغته الداخلية
  3. DeepSeek V4 Flash يعالج الطلب ويولد الرد
  4. SGLang يرد على Claude Code بنفس صيغة Anthropic API
  5. Claude Code يستخدم الأدوات المضمنة (Bash, Edit, Read...) لتنفيذ المهام على جهازك

لماذا لا يحتاج تسجيل دخول

المكونالدور في تجاوز تسجيل الدخول
ANTHROPIC_BASE_URL يغير وجهة API من api.anthropic.com إلى سيرفرك المحلي. كل طلب POST يذهب إلى سيرفرك بدلا من خوادم Anthropic.
ANTHROPIC_API_KEY أي قيمة وهمية تكفي. Claude Code يتحقق من وجود المفتاح فقط وليس من صحته. وجود المفتاح يمنع ظهور شاشة تسجيل الدخول.
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC يمنع أي طلب HTTP خارجي غير متعلق بالـ API (تليمتري، إحصائيات، استبيانات، تحديثات).

إعداد سيرفر SGLang

تشغيل DeepSeek V4 Flash على RHEL مع Podman

# تشغيل SGLang Container مع دعم Anthropic API
podman run --gpus all \
  --shm-size 32g \
  -p 30000:30000 \
  -v /path/to/models:/models:Z \
  --ipc=host \
  lmsysorg/sglang:latest \
  python3 -m sglang.launch_server \
    --model-path /models/DeepSeek-V4-Flash \
    --tp 4 \
    --host 0.0.0.0 \
    --port 30000 \
    --tool-call-parser deepseekv3
ملاحظة هامة: الـ flag --tool-call-parser deepseekv3 ضروري لدعم Claude Code في استدعاء الأدوات مثل Bash و Edit و Read. تأكد من إضافته.

اختبار السيرفر

# اختبار قائمة الموديلات
curl http://TBD:30000/v1/models

# اختبار الرسائل (Anthropic API مباشرة)
curl -X POST http://TBD:30000/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: sk-offline" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "DeepSeek-V4-Flash",
    "max_tokens": 100,
    "messages": [{"role": "user", "content": "Say hello"}]
  }'

استكشاف الأخطاء وإصلاحها

claude is not recognized (الأمر غير معروف)

افتح تيرمينال جديد. إذا استمرت المشكلة، أضف المسار يدويا:

# PowerShell:
$env:PATH = "$env:PATH;$env:USERPROFILE\.local\bin"

# CMD:
set PATH=%PATH%;%USERPROFILE%\.local\bin
There is an issue with the selected model (الموديل غير متاح)

سيرفر SGLang لا يعرف الموديل المطلوب. استخدم curl للتأكد من الموديلات المتاحة:

curl http://TBD:30000/v1/models

ثم استخدم أحد الأسماء الظاهرة مع --model.

connect ECONNREFUSED (لا يمكن الاتصال بالسيرفر)

سيرفر SGLang غير شغال أو IP/port غير صحيح. تحقق من:

  • أن السيرفر شغال: curl http://[IP]:30000/v1/models
  • صحة IP في ANTHROPIC_BASE_URL
  • الـ firewall يسمح بالاتصال على المنفذ 30000
Claude Code يطلب تسجيل دخول (login)

تأكد من:

  • وجود ANTHROPIC_BASE_URL وتشير إلى سيرفرك المحلي
  • وجود ANTHROPIC_API_KEY بقيمة وهمية
  • وجود CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1

إذا استمرت المشكلة، أضف DISABLE_LOGIN_COMMAND=1 واستخدم --bare.

النافذة تفتح وتقفل بسرعة (على Windows)

سيرفر SGLang غير متاح، لذا Claude Code يخرج مباشرة. حلول:

  • تأكد أن السيرفر شغال قبل تشغيل Claude Code
  • استخدم claude --bare -p "..." بدلا من الوضع التفاعلي
  • أو استخدم CMD بدلا من النقر المزدوج على ملف
/doctor ليس متاحا في هذه البيئة

أمر /doctor لا يعمل مع --bare. استخدمه في الوضع التفاعلي (بدون --bare).

الوكلاء (Agents)

Claude Code يدعم الوكلاء المتخصصين (sub-agents) الذين يمكن إنشاؤهم وتشغيلهم لإنجاز مهام محددة. جميع الوكلاء يعملون بشكل كامل أوفلاين.

الأوامر المتعلقة بالوكلاء

الأمرالوظيفة
/agentsيفتح قائمة الوكلاء المتاحين. يمكنك عرض وصف كل وكيل وتشغيله أو إنشاء وكيل جديد.
/forkيحول الجلسة الحالية إلى وكيل متخصص مع تعليمات مخصصة.

أنواع الوكلاء المضمنة

الوكلاء التاليين مثبتين مسبقا من السوق الرسمي ومخزنين محليا:

  • code-architect - تصميم بنية المشاريع وتحليل الكود الموجود
  • code-explorer - استكشاف قاعدة الكود وفهمها
  • code-reviewer - مراجعة الكود واقتراح التحسينات
  • legacy-analyst - تحليل الكود القديم
  • security-auditor - مراجعة أمنية للكود
  • test-engineer - كتابة واختبار الكود
  • code-simplifier - تبسيط الكود وتحسين readability
  • agent-creator - إنشاء وكلاء جدد
  • conversation-analyzer - تحليل المحادثات

يمكنك إنشاء وكلاء مخصصين بتعريفات خاصة بك عبر الأمر /agents.

تشغيل عدة وكلاء في نفس الوقت (Multi-Agent)

Claude Code يدعم تشغيل وكلاء متعددين بالتوازي، وجميعهم يشاركون نفس اتصال API مع سيرفر SGLang المحلي. لا يحتاجون أي اتصال إنترنت إضافي — كل الـ 9 وكلاء المضمنة والوكلاء المخصصين يعملون على نفس الموديل المحلي (DeepSeek V4 Flash).

الآلية

الخطوةالشرح
1. الجلسة الأم جلسة Claude Code الرئيسية هي نقطة التحكم. منها توزع المهام على الوكلاء.
2. /fork لتفويض مهمة استخدم /fork [تعليمة] داخل الجلسة لإنشاء وكيل فرعي في الخلفية يرث سياق المحادثة الحالية. الوكيل يعمل فورا وأنت تستمر في محادثتك الأصلية. النتيجة تعود إليك عند انتهائه.
3. /background لجلسة خلفية استخدم /background [تعليمة] لتحويل جلستك الحالية بكاملها إلى خلفية (وكيل خلفي) وتحرير التيرمينال لجلسة جديدة.
4. عدة وكلاء بالتوازي يمكنك تكرار /fork عدة مرات لتوزيع مهام مختلفة على وكلاء متعددين (مثلا: واحد يراجع الكود، واحد يكتب اختبارات، واحد يحلل الأمان). كل وكيل يستقبل طلباته الخاصة عبر API المحلي.
5. إدارة الوكلاء من تيرمينال آخر، استخدم claude agents لعرض جميع وكلاء الخلفية. الجلسات الخلفية تظهر أيضا في /resume مع علامة "bg".

مثال عملي: تطوير مميزة كاملة بتوزيع المهام

افترض أنك تريد إضافة نظام مصادقة (authentication) لمشروع:

# 1. افتح جلسة Claude Code رئيسية (مع ANTHROPIC_DEFAULT_HAIKU_MODEL للمهام الخلفية)
  set ANTHROPIC_BASE_URL=http://TBD:30000
  set ANTHROPIC_API_KEY=sk-offline
  set ANTHROPIC_DEFAULT_HAIKU_MODEL=DeepSeek-V4-Flash
  claude --model DeepSeek-V4-Flash

  # 2. داخل الجلسة، ابدأ بتوزيع المهام على وكلاء متخصصين:

  # وكيل لتصميم البنية (code-architect):
  /fork استخدم دورك كـ code-architect لتصميم بنية نظام مصادقة
  يدعم JWT و OAuth2. اكتب خطة في ملف AUTH-ARCH.md

  # وكيل لمراجعة الكود الموجود (code-reviewer) - بالتوازي:
  /fork استخدم دورك كـ code-reviewer وراجع جميع ملفات auth
  الموجودة في مجلد src/auth/. اكتب ملاحظاتك في CODE-REVIEW.md

  # يمكنك فتح وكيل ثالث في نفس الوقت:
  /fork استخدم دورك كـ security-auditor وحلل الثغرات الأمنية
  المحتملة في src/auth/. اكتب تقريرا في SECURITY.md

  # 3. استمر في عملك في الجلسة الرئيسية أثناء عمل الوكلاء في الخلفية
  # 4. عندما ينتهي كل وكيل، تظهر نتيجته في محادثتك تلقائيا
  # 5. راجع النتائج واطلب من test-engineer كتابة اختبارات:
  /fork استخدم دورك كـ test-engineer واكتب اختبارات وحدة
  لنظام المصادقة بناء على التصميم في AUTH-ARCH.md

إدارة وكلاء متعددين من تيرمينال منفصل

يمكنك فتح نافذة تيرمينال جديدة والتحكم بكل الوكلاء العاملين:

# عرض جميع الوكلاء النشطين وجلسات الخلفية
  claude agents

  # إرفاق بوكيل معين (attach)
  claude --resume <session-id>

  # إنهاء وكيل معين من داخل الجلسة
  /background stop

ملاحظات هامة للتشغيل الأوفلاين متعدد الوكلاء

  • جميع الوكلاء يشاركون نفس API: كل وكيل يرسل طلباته إلى نفس سيرفر SGLang المحلي — لا يوجد اتصال خارجي.
  • كل وكيل له سياقه المستقل: سياق كل وكيل منفصل تماما، مما يعني أن /fork يعطي كل وكيل نسخة من المحادثة عند نقطة التفويض.
  • تأثير على أداء السيرفر: مع عدة وكلاء نشطين، سيرسلون طلبات متزامنة إلى SGLang. إذا لاحظت بطء، قلل عدد الوكلاء المتزامنين أو استخدم /compact لتقليل طول السياق.
  • الوكلاء لا يستهلكون رصيد API: بما أن السيرفر محلي ولا توجد رسوم، يمكنك تشغيل أي عدد تريده من الوكلاء دون قيود.
  • الوكلاء يعملون بدون --bare: يمكن للوكلاء استخدام جميع الأدوات (Bash, Edit, Read, Glob, Grep, TodoWrite, LS) لأنهم يعملون في جلسات كاملة وليس في وضع --bare.
  • يجب ضبط ANTHROPIC_DEFAULT_HAIKU_MODEL: بدون هذا المتغير، المهام الخلفية (تسمية الوكلاء، تلخيص الجلسات) تحاول استخدام موديل Haiku من خوادم Anthropic وتفشل. اضبطه على نفس اسم موديل DeepSeek المحلي لضمان عمل جميع الوكلاء بشكل كامل أوفلاين.
خلاصة: ميزة تعدد الوكلاء تعمل بكامل طاقتها أوفلاين. يمكنك تفويض مهام متعددة لـ 10 وكلاء بالتوازي، وجميعهم يرسلون طلباتهم إلى سيرفر SGLang المحلي دون الحاجة لأي اتصال إنترنت أو حساب مدفوع.

الوكلاء المتعددون = فريق تطوير كامل

من أقوى مزايا التشغيل الأوفلاين متعدد الوكلاء هي القدرة على محاكاة فريق تطوير كامل يعمل بالتوازي. كل /fork ينشئ عاملا مستقلا بسياقه وأدواته ومهمته الخاصة.

  أنت (صاحب الجلسة / Product Owner)
    |
    |-- /fork "code-architect"          (معماري - يصمم الهيكل)
    |     يكتب AUTH-ARCH.md, يعرف المكونات وتدفق البيانات
    |
    |-- /fork "code-reviewer"           (مطور أول - يراجع الكود)
    |     يحلل src/auth/, يكتب CODE-REVIEW.md
    |
    |-- /fork "security-auditor"        (مهندس أمن - يدقق)
    |     يفحص الثغرات, يكتب SECURITY.md
    |
    |-- /fork "test-engineer"           (مهندس اختبارات)
    |     ينشئ اختبارات وحدة و integration
    |
    |-- /fork "legacy-analyst"          (محلل - يدرس الكود القديم)
    |     يوثق الأنماط القديمة ومسارات الترحيل
    |
    أنت تواصل العمل في الجلسة الرئيسية وكل الوكلاء الخمسة يعملون بالتزامن.
    كل واحد يرجع نتيجته تلقائيا عندما ينتهي.

كل هؤلاء الوكلاء يعملون في نفس الوقت على نفس الموديل المحلي الواحد (DeepSeek V4 Flash). لا ينتظرون بعضهم، لا يوقفون جلستك الرئيسية، ولا يكلفون شيئا. أنت تتوسع من مطور واحد إلى فريق كامل بمجرد أوامر /fork.

كيف يعمل هذا بموديل واحد؟ الموديل المحلي يعالج طلبات وكيل واحد في كل مرة، لكن التبديل بين الوكلاء سريع جدا لدرجة أنهم يبدون وكأنهم يعملون بالتوازي. مع SGLang و CUDA 13، الـ batching الفعال يتعامل مع عدة طلبات متزامنة بسلاسة.

التوثيق الرسمي لـ Claude Code يؤكد هذا السلوك:

الاستنتاجات الرئيسية:

  1. Fork subagents (/fork): يرثون نفس الموديل بالضبط من جلسة الأم. كل طلب من /fork يذهب إلى نفس ANTHROPIC_BASE_URL بنفس اسم الموديل.
  2. Named subagents (الـ 9 المضمنة): كل واحد يمكنه تحديد موديل مختلف في ملف تعريفه، لكن في إعدادك الأوفلاين يوجد موديل واحد فقط على SGLang (DeepSeek V4 Flash). أي اسم موديل ترسله إلى السيرفر المحلي يتحل إلى الموديل المحمّل. ANTHROPIC_BASE_URL يوجه كل الحركة — من الجلسة الأم ومن كل وكيل — إلى نفس سيرفر SGLang المحلي.
  3. CLAUDE_CODE_FORK_SUBAGENT=1: هذا المتغير يفعل مشاركة prompt cache عبر الأطفال المتوازيين، مما يقلل التوكينز حتى 90% للأطفال 2-N. لا يغير الموديل.
  4. مع 4x H100 94GB و CUDA 13: SGLang يتعامل مع الطلبات المتزامنة بكفاءة عبر CUDA batching. يمكن تشغيل 10-20+ وكيل بالتوازي على هذا الجهاز.

الخلاصة: موديل واحد (DeepSeek V4 Flash) يكفي لتشغيل فريق كامل — جلسة رئيسية + 9 وكلاء مضمنة + وكلاء مخصصة بلا عدد + /fork غير محدود. التوثيق الرسمي يؤكد هذه المعمارية.

فحص أخطاء المتصفح (MCP Servers)

Claude Code يمكنه الاتصال بمتصفحك عبر MCP servers لقراءة console logs, network requests, أخطاء JavaScript, وهيكل الصفحة — كلها كنص, لا تحتاج رؤية. هذا الحل مثالي للتشغيل الأوفلاين حيث الموديل لا يدعم الصور.

الموصى به: Playwright MCP (Microsoft)

مخدم MCP الرسمي من Microsoft لأتمتة المتصفح مع فحص console و network:

الأداةالوصفالنوعيحتاج رؤية؟
browser_console_messagesمخرجات console (log, warn, error, info, debug)نصلا
browser_network_requestsطلبات network مع method, URL, status code, timingJSONلا
browser_snapshotAccessibility tree — هيكل الصفحة كنصنصلا

npx @playwright/mcp
claude mcp add --transport stdio playwright -- npx @playwright/mcp --headless

بديل: agent-browse (tollebrandon)

التقاط مهيكل أكثر للأخطاء مع أدوات مخصصة:

npx agent-browser

كيف تعمل أدوات المتصفح

  1. تبدأ جلسة متصفح ثابتة (تستمر عبر أوامر متعددة)
  2. تتنقل إلى الـ URL الذي تطلبه
  3. تلتقط مخرجات console, network requests, والأخطاء في الوقت الحقيقي
  4. يمكنك الضغط على أزرار, ملء فورمات, كتابة نصوص — ثم فحص النتائج مرة أخرى

جميع مخدمات MCP تعمل أوفلاين بنفس الطريقة: تشتغل على جهازك المحلي, كل طلبات AI تذهب إلى SGLang عبر ANTHROPIC_BASE_URL.

كيف يختار Claude الأداة المناسبة

Claude Code يتبع ترتيب أولويات عندما تطلب منه مهمة متصفح:

ملف تعليمات المشروع (CLAUDE.md)

CLAUDE.md هو ملف Markdown في جذر مشروعك يعمل كمجموعة تعليمات دائمة لـ Claude Code. كلما فتحت جلسة في مجلد المشروع، يقرأ Claude هذا الملف ويستخدمه كسياق لفهم conventions مشروعك ومعمارية وتفضيلاتك.

اعتبرها system prompt خاصة بكل مشروع تتحكم فيها أنت. هي الطريقة الأكثر فعالية لجعل Claude Code يعمل بالطريقة التي تريدها دون تكرار التعليمات كل جلسة.

كيف يعمل

  • ضع CLAUDE.md في جذر مشروعك (بجانب package.json, Cargo.toml, go.mod إلخ).
  • Claude Code يقرأه تلقائيا عند بدء جلسة في ذلك المجلد.
  • محتوى الملف يحقن في system prompt، لذا Claude يتبعه طوال الجلسة.
  • يستمر عبر /clear (الجلسات الجديدة في نفس المشروع تقرأه مجددا).
  • التغييرات على CLAUDE.md تطبق عند بدء الجلسة التالية.
  • استخدم /init لإنشائه تفاعليا، أو أنشئه يدويا.
  • إذا كان CLAUDE_CODE_NEW_INIT=1، يفتح وضعا تفاعليا متقدما لإنشاء الملفات الشخصية والـ skills و hooks.

ماذا تضع في CLAUDE.md

التصنيفأمثلة
التقنيات المستخدمة اللغات، الفريموركات، إصدارات runtime، مدير الحزم
قواعد الكود أسلوب التسمية، هيكل الملفات، قواعد الـ import، formatting
متطلبات الاختبارات إطار الاختبارات، تغطية متوقعة، أين توجد الاختبارات
أوامر البناء والتشغيل كيف تبني، تختبر، تشغل linter، تنشر
هيكل المشروع الغرض من كل مجلد، الملفات الرئيسية، قرارات معمارية
التفضيلات المكتبات المفضلة، الأنماط الممنوعة، أسلوب الكود
ملاحظات الأوفلاين API المحلي، اسم الموديل، متغيرات البيئة المطلوبة

نصائح لأفضل CLAUDE.md

  • كن محددا، لا عام. بدل "اكتب كود نظيف"، قل "استخدم early returns، أقصى 80 حرفا لكل سطر، PascalCase للـ types".
  • أضف الأوامر الدقيقة. npm run test، cargo build --release، python -m pytest tests/ — حتى يتمكن Claude من تشغيلها.
  • خليها موجزة. 50-150 سطرا مثالياً. الكثير من السياق يشتت التركيز.
  • حدثها مع تطور المشروع. CLAUDE.md قديم يضر أكثر مما ينفع.
  • استخدم /init مع CLAUDE_CODE_NEW_INIT=1 للوضع التفاعلي المتقدم.

مثال CLAUDE.md

# Project: auth-service

  ## Tech Stack
  - Go 1.23, PostgreSQL 16, Redis 7
  - Framework: chi router, sqlx for DB, testify for tests
  - Build: go build ./cmd/server
  - Test: go test ./... -race -count=1

  ## Code Conventions
  - Error handling: always wrap errors with context (fmt.Errorf("fetch user %d: %w", id, err))
  - Logging: use structured logger (zerolog), no fmt.Print
  - HTTP handlers: separate handler + service + repository layers
  - SQL queries: use sqlx named params :user_id not $1

  ## Testing
  - Unit tests in _test.go next to source files
  - Integration tests in tests/integration/ with //go:build integration tag
  - Run: make test (unit), make test-integration (integration)
  - Aim for 80%+ coverage on business logic

  ## API Design
  - RESTful, JSON request/response
  - Auth: Bearer JWT tokens
  - All endpoints prefixed with /api/v1/

  ## Project Structure
  - cmd/server/     -- main entrypoint
  - internal/handler/   -- HTTP handlers
  - internal/service/  -- business logic
  - internal/repository/   -- data access
  - migrations/     -- SQL migrations (goose)

الفرق بين CLAUDE.md و settings.json

الملفالنطاقالغرض
CLAUDE.md كل مشروع على حدة تعليمات، conventions، أوامر — كيف يتصرف Claude في هذا المشروع بالذات
~/.claude/settings.json عام / المستخدم كله متغيرات البيئة، الصلاحيات، إعدادات API، خيارات الميزات
~/.claude/projects/<project>/claude.json إعدادات خاصة بمشروع تجاوزات (overrides) لـ settings.json خاصة بمشروع معين

الملفات الثلاثة تعمل معاً. settings.json يتحكم في كيفية اتصال Claude Code (API endpoint, المفاتيح, التليمتري). CLAUDE.md يتحكم في ماذا يفعل Claude بعد الاتصال (أسلوب الكود, الأوامر, conventions).

نصيحة للمشاريع المتعددة

يمكن أن يكون لديك ملفات CLAUDE.md مختلفة في كل مشروع. افتح جلسة في المشروع A و Claude يتبع conventions المشروع A. استخدم /cd <path> لنقل جلستك إلى المشروع B، و Claude يتكيف مع CLAUDE.md الخاص بذلك المشروع في منتصف الجلسة (يتطلب v2.1.169+).

ملخص التشغيل الأوفلاين

Claude Code يعمل بكامل طاقته أوفلاين
  • جميع slash commands تعمل (ما عدا WebFetch و WebSearch اللذان يحتاجان إنترنت)
  • الوكلاء (Agents) يعملون عبر API السيرفر المحلي
  • أدوات Bash, Edit, Read, Glob, Grep تعمل محليا
  • لا يحتاج أي حساب أو تسجيل دخول
  • لا يحتاج أي اتصال بالإنترنت
ما لا يعمل بدون إنترنت
  • WebFetch - جلب محتوى من URL
  • WebSearch - البحث في الويب
  • إضافات MCP السحابية (GitHub, GitLab, Linear, Discord) - تحتاج اتصالا بخدماتها

MCP المحلي (مثل Playwright MCP, agent-browser, agent-browse) يعمل 100% أوفلاين لأنه يشتغل على جهازك ويتواصل مع سيرفر SGLang المحلي.

هذه الميزات ليست ضرورية لعمل Claude Code كمحرر كود ذكي.

طريقة التشغيل الكامل أوفلاين (بدون إنترنت)

ما تحتاجه

  • سيرفر SGLang على RHEL 10 (أو أي Linux) يشغل DeepSeek V4 Flash
  • سيرفر SGLang لازم يشغل /v1/messages endpoint (PR #18630 - مدعوم أصلاً)
  • اتصال شبكة محلية (LAN) بين جهازك وسيرفر SGLang
  • ملفات Claude Code من هذه الحزمة (تنقل عبر USB للجهاز المعزول)
مهم: SGLang يدعم Anthropic Messages API (/v1/messages) بشكل أصلي. لا تحتاج أي Proxy إضافي (مثل LiteLLM). كل شيء مباشر بين Claude Code و SGLang.

الخطوات

1

تأكد أن سيرفر SGLang شغال

من أي جهاز في الشبكة، اختبر الاتصال:

curl http://[IP-SERVER]:30000/v1/models

يجب أن ترى قائمة بالموديلات المتاحة.

2

ثبّت Claude Code على جهازك

اختر نظام التشغيل:

خيار 1 - التثبيت التلقائي (كمسؤول):

.\windows\install-offline.bat

خيار 2 - التثبيت اليدوي:

mkdir -Force "$env:USERPROFILE\.local\bin"
copy windows\claude.exe "$env:USERPROFILE\.local\bin\"
$userPath = [Environment]::GetEnvironmentVariable("PATH", "User")
if ($userPath -notlike "*\.local\bin*") {
  [Environment]::SetEnvironmentVariable("PATH", "$userPath;$env:USERPROFILE\.local\bin", "User")
}
mkdir -Force "$env:USERPROFILE\.claude"
copy config\settings.json "$env:USERPROFILE\.claude\"

خيار 1 - التثبيت التلقائي:

chmod +x linux/install-offline.sh
./linux/install-offline.sh

خيار 2 - التثبيت اليدوي:

mkdir -p ~/.local/bin
cp linux/claude ~/.local/bin/
chmod +x ~/.local/bin/claude
export PATH="$HOME/.local/bin:$PATH"
mkdir -p ~/.claude
cp config/settings.json ~/.claude/settings.json
3

عدّل IP السيرفر في الإعدادات

افتح الملف ~/.claude/settings.json وغيّر [IP-SERVER] إلى IP سيرفر SGLang الحقيقي:

{
  "env": {
    "ANTHROPIC_BASE_URL": "http://TBD:30000",
    "ANTHROPIC_API_KEY": "sk-offline",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "DISABLE_AUTOUPDATER": "1"
  }
}

أو استخدم متغيرات البيئة مباشرة (بدون تعديل ملف):

4

شغّل Claude Code

التفاعلي (REPL) - الأفضل:

set ANTHROPIC_API_KEY=sk-offline
set CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1

$env:ANTHROPIC_API_KEY="sk-offline"
$env:CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC="1"

لماذا يعمل بدون تسجيل دخول؟

المتغير / الخيارالتفسير
ANTHROPIC_BASE_URL يوجه جميع طلبات API من api.anthropic.com إلى سيرفر SGLang المحلي. هذا هو المفتاح الأساسي.
ANTHROPIC_API_KEY مفتاح API وهمي (مثلاً sk-offline). SGLang يتجاهله ولا يتحقق منه. يمنع Claude Code من طلب تسجيل دخول لأن المفتاح موجود.
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1 يمنع جميع الاتصالات غير الأساسية: التليمتري، تقارير الأخطاء، التحديثات، الاستبيانات، إلخ.
DISABLE_AUTOUPDATER=1 يمنع التحديثات التلقائية (لأن ما في إنترنت أصلاً).

ماذا عن --bare و CLAUDE_CODE_SIMPLE؟

--bare و CLAUDE_CODE_SIMPLE هي خيارات إضافية لتعطيل ميزات متقدمة (مثل hooks, LSP, plugins, keychain). ليست ضرورية لتجاوز تسجيل الدخول. المفاتيح الأساسية هي ANTHROPIC_BASE_URL و ANTHROPIC_API_KEY فقط.

الخلاصة: كل طلبات Claude Code تذهب إلى سيرفر SGLang المحلي فقط. لا شيء يغادر شبكتك المحلية. لا يحتاج أي اتصال بالإنترنت. لا يحتاج أي حساب أو تسجيل دخول.

إعداد سيرفر SGLang (DeepSeek V4 Flash)

تشغيل SGLang Container على RHEL (Podman)

# تشغيل DeepSeek V4 Flash مع دعم Anthropic API
podman run --gpus all \
  --shm-size 32g \
  -p 30000:30000 \
  -v /path/to/models:/models:Z \
  --ipc=host \
  lmsysorg/sglang:latest \
  python3 -m sglang.launch_server \
    --model-path /models/DeepSeek-V4-Flash \
    --tp 4 \
    --moe-runner-backend flashinfer_mxfp4 \
    --enable-deepseek-v4-fp4-indexer \
    --tool-call-parser deepseekv3 \
    --host 0.0.0.0 \
    --port 30000

اختبار الاتصال بـ Anthropic API

# اختبار قائمة الموديلات
curl http://TBD:30000/v1/models

# اختبار /v1/messages (Anthropic API مباشر)
curl -X POST http://TBD:30000/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: sk-offline" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "DeepSeek-V4-Flash",
    "max_tokens": 100,
    "messages": [{"role": "user", "content": "Say hello"}]
  }'
ملاحظة: الـ --tool-call-parser deepseekv3 ضروري لدعم Claude Code في استدعاء الأدوات (Bash, Edit, Read). تأكد من إضافته عند تشغيل SGLang.

استكشاف الأخطاء وإصلاحها

المشكلة: "claude is not recognized" أو الأمر غير معروف

الحل: افتح terminal جديد. أو أضف المسار يدويًا:

# PowerShell:
$env:PATH = "$env:PATH;$env:USERPROFILE\.local\bin"

# CMD:
set PATH=%PATH%;%USERPROFILE%\.local\bin
المشكلة: "There's an issue with the selected model (...) It may not exist"

الحل: سيرفر SGLang ما عنده الموديل. تأكد أن --model-path صحيح وأن الموديل مضبوط في SGLang. استخدم curl /v1/models لتتأكد من الموديلات المتاحة.

المشكلة: "connect ECONNREFUSED" أو "socket hang up"

الحل: سيرفر SGLang مو شغال أو الـ IP/port غلط. تأكد من:

  • السيرفر شغال: curl http://[IP]:30000/v1/models
  • الـ IP صحيح في ANTHROPIC_BASE_URL
  • الـ Firewall يسمح بالاتصال على port 30000
المشكلة: يطلب تسجيل دخول (Login)

الحل: تأكد من أن:

  • ANTHROPIC_BASE_URL مضبوط على سيرفرك المحلي
  • ANTHROPIC_API_KEY موجود (أي قيمة وهمية)
  • CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
المشكلة: النافذة تفتح وتقفل بسرعة

الحل: السيرفر مو شغال. اشتغل على السيرفر أولاً أو استخدم claude --bare -p "..." (one-shot) بدلاً من REPL.
